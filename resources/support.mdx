## Reporting Issues

### Bug Reports

Found a bug? Report it on GitHub Issues:

<Card title="Report a Bug" icon="bug" href="https://github.com/NeoRazorX/facturascripts/issues/new">
  Create a detailed bug report
</Card>

When reporting a bug, include:
- **FacturaScripts version** - Check in the about section
- **PHP version** - Run `php -v`
- **Database type and version** - MySQL, MariaDB, or PostgreSQL
- **Web server** - Apache, Nginx, or built-in PHP server
- **Operating system** - Linux, Windows, macOS
- **Steps to reproduce** - Detailed instructions
- **Expected behavior** - What should happen
- **Actual behavior** - What actually happens
- **Error messages** - Full error text and stack traces
- **Screenshots** - If applicable

## Security Vulnerabilities

<Warning>
  **Do not report security vulnerabilities through public GitHub issues!**
</Warning>

If you discover a security vulnerability, please email:

<Card title="Security Contact" icon="shield" href="mailto:carlos@facturascripts.com">
  carlos@facturascripts.com
</Card>

Provide details about:
- The type of vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes

We take security seriously and will respond promptly.

## Before Asking for Help

To get the fastest resolution:

1. **Search existing resources**
   - Check the [FAQ](/resources/faq)
   - Review [troubleshooting guide](/resources/troubleshooting)
   - Search GitHub Issues for similar problems

2. **Gather information**
   - Note your FacturaScripts version
   - Check error logs
   - Document steps to reproduce

3. **Choose the right channel**
   - Discord - Quick questions and community help
   - GitHub Issues - Bug reports and feature requests
   - Email - Security vulnerabilities
   - Contact form - Business inquiries

4. **Provide details**
   - Include version information
   - Share error messages
   - Describe what you've already tried
   - Be clear and concise

## Response Times

<Info>
  FacturaScripts is an open-source project maintained by volunteers. Response times may vary.
</Info>

- **Discord** - Usually within hours during business days
- **GitHub Issues** - Within 1-3 business days
- **Security vulnerabilities** - Prioritized, typically within 24 hours
- **Contact form** - Within 3-5 business days

## Community Guidelines

When seeking support:
- Be respectful and patient
- Search before asking duplicate questions
- Provide complete information
- Follow up with solutions you find
- Help others when you can
- Give back to the community
